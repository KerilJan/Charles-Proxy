Используя Charles Proxy, я тестировал корзину интернет-магазина.

Ниже представлены видео, в которых я изменяю данные с помощью прокси-сервера для сайта в Google Chrome на MacBook Air:

- [Изменение количества товаров в корзине](https://drive.google.com/drive/folders/1cbbqLpVHoH7VK9RuOYByTBuEBmvjIngf) — запрос отправляет 2 товара, а ответ возвращает 500.
- [Изменение статуса ответа с 200 на 403](https://drive.google.com/drive/folders/1ZF36RNlZAKy-XNBO5NmGEo6NPe1Y6pVA) - 
- [Перенаправление запроса с Prod на QA](https://drive.google.com/drive/folders/1jAN6buoxiO9oPrDiDSKgPoSzKe202YtP)

Также я изменял данные с помощью прокси-сервера для сайта в Google Chrome на iPhone:

- [Удаление товаров из корзины](https://drive.google.com/file/d/1cTA2vnoyhpx84qfBvifchAAeeIByy6eR/view?usp=sharing) - Задание: Добавьте несколько разных товаров в корзину, отправьте запрос на удаление любого из них из корзины, но видоизмените его так, чтобы он удалил другой товар,
- [Замена сайта на изображение](https://drive.google.com/drive/folders/1lZ66948WujZD1wmM485geU0DVefzgytY) - Задание: Смоделируйте ситуацию, при которой при обращении к https://intern.demoshopping.ru пользователь увидит в браузере любую картинку.
- [Поиск user-agent в заголовках](https://drive.google.com/file/d/1Irh2bT0iRU2m7SYh4DJh_yjwStBB_tn7/view?usp=drive_link) - Задание: Приложите скриншот любого перехваченного HTTPs-запроса с вашего мобильного устройства. В header user-agent должна быть информация о вашем устройстве.
